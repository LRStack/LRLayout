Fixed column count

Set --cols to a number for a fixed-column grid that shares width evenly — three columns regardless of width. Unlike auto-fit, a fixed count can't drop columns, so below --cols × --min the tracks would overflow; wrapping the grid in .lr-scroll-x (or lowering --min) lets it scroll instead of bleeding past the edge. Narrow the window to see it scroll.

Aligning items in their tracks

Items stretch to fill their cell by default, so every tile in a row matches the tallest. .lr-items-start sets --items (align-items) so each keeps its natural height instead.

auto-fit vs auto-fill

With only a couple of items in a wide container, .lr-cols-fit (the default) collapses the empty tracks so the items stretch to fill the row; .lr-cols-fill keeps them, leaving equal trailing columns.
