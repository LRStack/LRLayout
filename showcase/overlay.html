overlay

Floats one child over the region's other content — a play button on a frame, a badge on an avatar, a dialog over a backdrop. Where frame and cover center content that stays in flow, overlay takes a child out of flow and positions it on top; the rest of the region is undisturbed. (The classic imposter of intrinsic-layout lore.)

A play button over media

The host is any element — here a frame with lr-overlay added. The image fills the frame as usual; the .lr-overlay-item button is lifted out of flow and centered on top, so it never disturbs the media. Default placement is dead center, both axes.

A badge pinned to a corner

Add a placement modifier to leave the center. Each modifier drops the constraint on one edge, pulling the item to the other; lr-overlay-top + lr-overlay-end together make a corner. --inset (set on the item) holds it a hair off the edge.

Every placement

With no modifier the item centers. The four edge modifiers (top, bottom, start, end) pin to an edge; any two non-opposite ones compose a corner. All nine items below share one host — they're all .lr-overlay-item, differing only by modifier. Edges and corners are logical, so they flip under RTL.

A dialog that can't overflow

Add lr-overlay-contain and the item caps itself to the host (minus --inset on every side) and scrolls internally — so a tall dialog stays parked inside the region instead of bleeding past its edges. The backdrop is just the host's own content, sitting in normal flow underneath.

The auto-margin centering still applies on the axis that fits, so a short dialog would sit dead center; only the overflowing axis is clamped.

And because the cap is 100% minus --inset, the same knob that offsets a corner badge also sets the dialog's breathing room here.
