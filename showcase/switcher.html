switcher

A small set of equal-width items that share one row, then flip — all together, all at once — to a full-width stacked column when the container drops below --threshold. No media queries: the switch rides a flexbox quantity query (flex-basis: calc()).

Three peers, all-or-nothing

A plain .lr-switcher. Wide enough, the three panels share the row as equal columns; narrow the window past --threshold and they switch together to a full column — never a half-wrapped in-between. Compare with cluster, which wraps one item at a time.

Tune the switch point

Identical markup — only --threshold changes. A higher threshold flips to a column sooner (at a wider container). It's an approximate switch width: the gap isn't subtracted from the math, so the real flip lands a touch wider than the value set.

Two side-by-side panels

The other common case — a pair of equal panels that stack on narrow screens. Each panel is itself an .lr-stack; --items: stretch (the default) keeps them equal height while side by side.

Two zones competing for one row. Plenty of room, so they sit side by side as equal columns.

After

Squeeze the container and both drop to full width at the same instant — no lopsided wrap.

When to reach for grid instead

switcher is for a small number of peers and has no count limit by design. For an arbitrary-length list of cards, use grid with --min — it reflows the column count a row at a time, instead of forcing every item full-width together.
